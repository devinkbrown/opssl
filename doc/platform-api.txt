opssl PLATFORM API REFERENCE
=============================

This document describes the platform abstraction layer in opssl/platform.h.

opssl is part of the libop ecosystem. General allocation, strings, and data
structures come from libop (op_malloc, op_free, op_strbuf_t, etc.). This
header adds what TLS specifically needs on top: secure key memory,
constant-time operations, entropy, endian helpers, and CPU feature detection.


SECURE KEY MEMORY
=================

    void *opssl_key_alloc(size_t size);
    void *opssl_key_realloc(void *ptr, size_t old_size, size_t new_size);
    void  opssl_key_free(void *ptr, size_t size);

For key material ONLY. These add:
- mlock to prevent swapping to disk
- Guaranteed zeroing on free (explicit_bzero before munlock)

General allocations should use op_malloc/op_free from libop. Using
opssl_key_alloc for non-key data wastes locked memory.


SECURE ERASURE
==============

    void opssl_memzero(void *ptr, size_t len);

Guaranteed not to be optimized away by the compiler. Uses:
- explicit_bzero (glibc, FreeBSD)
- memset_s (C11 Annex K)
- Volatile write barrier (fallback)

All key material in opssl is zeroed via this function before free.


CONSTANT-TIME OPERATIONS
=========================

    int opssl_ct_eq(const void *a, const void *b, size_t len);

Constant-time byte comparison. Returns 1 if all bytes are equal,
0 otherwise. Execution time depends only on len, never on content.
Used for HMAC tag verification, password comparison, etc.

    int opssl_ct_is_zero(const void *buf, size_t len);

Returns 1 if all bytes in buf are zero, 0 otherwise. Constant-time.

    int opssl_ct_word_is_zero(uint64_t x);

Returns 1 if x is zero, 0 otherwise. Constant-time (no branch).

    uint64_t opssl_ct_mask(int condition);

Returns 0xFFFFFFFFFFFFFFFF if condition is non-zero, 0 otherwise.
Used for branchless conditional selection in crypto code.

    void opssl_ct_select(void *dst, const void *a, const void *b,
                         size_t len, int select_a);

Constant-time conditional copy. If select_a is non-zero, copies a to
dst; otherwise copies b to dst. No branch on the condition.

    size_t opssl_ct_min(size_t a, size_t b);

Constant-time minimum. Returns the smaller value without branching.


ENTROPY / CSPRNG
================

    int opssl_random_bytes(void *buf, size_t len);

Fill buf with len bytes of cryptographically secure random data.
Uses the best available source:
1. getrandom(2) (Linux 3.17+)
2. getentropy(3) (OpenBSD, macOS)
3. arc4random_buf(3) (fallback)

Returns 1 on success, 0 on failure (entropy source unavailable).

    int opssl_random_uniform(uint32_t upper_bound, uint32_t *out);

Generate a uniformly distributed random number in [0, upper_bound).
Uses rejection sampling to avoid modulo bias. Returns 1 on success.

    int  opssl_random_init(void);
    void opssl_random_cleanup(void);

Initialize/cleanup the CSPRNG subsystem. Called automatically by
opssl_init/opssl_cleanup. Not normally called directly.


ENDIAN / BYTE MANIPULATION
===========================

Network byte order (big-endian) read functions:

    uint16_t opssl_be16(const uint8_t *p);
    uint32_t opssl_be32(const uint8_t *p);
    uint64_t opssl_be64(const uint8_t *p);

Network byte order write functions:

    void opssl_put_be16(uint8_t *p, uint16_t v);
    void opssl_put_be32(uint8_t *p, uint32_t v);
    void opssl_put_be64(uint8_t *p, uint64_t v);

All are inline functions with no alignment requirements — they work
on any byte pointer.


HARDWARE FEATURE DETECTION
===========================

    bool opssl_has_aesni(void);

Returns true if the CPU supports AES-NI instructions. When true, opssl
uses AES-NI for AES operations (aes_ni.c) instead of the software
implementation (aes.c).

    bool opssl_has_pclmul(void);

Returns true if the CPU supports PCLMULQDQ (carry-less multiplication).
Used for GCM GHASH acceleration alongside AES-NI.

These check CPUID on x86_64. On other architectures they return false
and the software implementations are used.
